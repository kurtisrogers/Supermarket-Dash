# Supermarket Dash

Compare your UK shopping list across all major supermarkets by price — including **Nectar**, **Clubcard**, **More Card**, and other loyalty schemes. See the cheapest single shop, or follow the **savings map** to split your shop across multiple stores for maximum savings.

**Live demo:** Enable GitHub Pages on this repo (Settings → Pages → Source: GitHub Actions).

## Features

- Compare prices across **8 UK supermarkets**: Tesco, Sainsbury's, Asda, Morrisons, Aldi, Lidl, Waitrose, Ocado
- Toggle your **loyalty cards** to see member prices (Nectar, Clubcard, More Card, myWaitrose, Lidl Plus)
- **Single-store ranking** — find the cheapest place to buy everything in one trip
- **Savings map** — optimal multi-store plan showing which items to buy where
- **Item-by-item breakdown** with loyalty price indicators
- **Basket helpers** per supermarket — copy formatted lists, open store websites, step-by-step guides
- **Daily price updates** via GitHub Actions cron

## Tech stack

- **Node.js** — build tooling and price update scripts
- **Pico CSS** — minimal, accessible styling
- **htmx** — loads per-supermarket basket guide partials
- **Alpine.js** — reactive shopping list and comparison UI
- **GitHub Pages** — static hosting (no backend required at runtime)

## Daily price updates

A GitHub Actions workflow runs **every day at 06:00 UTC** (`.github/workflows/update-prices.yml`):

1. Runs `npm run update-prices`
2. Rebuilds the site
3. Commits changes if prices changed

### Live prices (optional)

By default, the daily cron **fetches live product catalogs** from each supermarket's website API and writes them to `src/data/catalogs/{store}.seed.json`.

- **Tesco** and **Sainsbury's** are fetched directly (no API key needed)
- **Asda, Morrisons, Waitrose** require `PEPESTO_API_KEY` in GitHub Secrets
- **Aldi, Lidl, Ocado** have no accessible public API — existing seed data is kept

Optional secrets: `PEPESTO_API_KEY`, `TESCO_API_KEY` (if Tesco rotates their public key)

## Product catalogue

Each supermarket has its own catalog file under `src/data/catalogs/`:

| File | Purpose |
|------|---------|
| `{store}.seed.json` | Curated starter catalog (~50 products per store, branded + own-label) |
| `{store}.json` | Full live catalog from Pepesto (generated when `PEPESTO_API_KEY` is set) |

The daily update merges all per-store catalogs into `src/data/products.json` for search and comparison.

### ALL products from each supermarket

Each supermarket has a `{store}.seed.json` file that is **fetched live** and updated by the daily cron:

| Store | Fetch method |
|-------|--------------|
| Tesco | Tesco GraphQL API (`xapi.tesco.com`) — category browse + keyword discovery |
| Sainsbury's | Sainsbury's GOL API — category tree + keyword discovery |
| Asda, Morrisons, Waitrose | Pepesto API (requires `PEPESTO_API_KEY`) |
| Aldi, Lidl, Ocado | No public API — seed retained until a source is added |

```bash
# Fetch one store into its seed file
npm run update-store -- tesco

# Fetch all stores, merge, and write products.json
npm run fetch-catalogs

# Limit pages per category/search (for testing)
FETCH_MAX_PAGES=5 npm run update-store -- tesco
```

The daily cron runs `npm run update-prices`, which loops each supermarket, writes `{store}.seed.json`, then merges into `products.json`.

Search works by **name**, **brand**, **barcode**, or **store SKU**. On mobile, tap **📷 Scan** to use your camera — Chrome on Android uses the native Barcode Detector API, with an html5-qrcode fallback elsewhere.

To add more seed products, edit `src/data/catalog-definitions.js` and run `npm run generate-catalogs`.

## Basket automation

UK supermarkets **do not expose public add-to-basket APIs** for third-party apps. Supermarket Dash provides the fastest practical workflow per store:

| Store | Method |
|-------|--------|
| Tesco, Sainsbury's, Asda, Morrisons, Waitrose, Ocado | Copy list → open store → search & add manually |
| Aldi, Lidl | Copy/print list → shop in-store |

Each store has a detailed **Basket help** guide loaded via htmx.

## Project structure

```
src/
  data/
    catalogs/            # Per-supermarket catalogs ({store}.seed.json + live {store}.json)
    catalog-definitions.js  # Templates for seed generation
    products.seed.json   # Merged catalogue snapshot
    products.json        # Generated by update-prices (committed)
    supermarkets.json    # Store config, loyalty schemes, URLs
  lib/
    catalog-merge.js     # Merge per-store catalogs for comparison
    catalog-load.js      # Load live or seed catalog per store
    pepesto-catalog.js   # Map Pepesto API responses
    search.js            # Name, brand, SKU, and barcode search
    barcode.js           # Camera barcode scanning
  partials/basket/       # htmx basket guide fragments
  css/custom.css
  index.html
scripts/
  generate-store-seeds.js  # Build per-store seed files from templates
  build-catalog.js         # Merge catalogs into products.json
  update-prices.js         # Daily update (Pepesto full catalog or seeds)
  build.js                 # Builds docs/ for GitHub Pages
  dev-server.js            # Local development server
docs/                    # Built static site (deployed to Pages)
.github/workflows/
  update-prices.yml      # Daily cron
  deploy.yml             # GitHub Pages deployment
```

## Deploy to GitHub Pages

1. Push to `main`
2. Go to **Settings → Pages → Build and deployment → Source: GitHub Actions**
3. The deploy workflow runs automatically on every push to `main`
